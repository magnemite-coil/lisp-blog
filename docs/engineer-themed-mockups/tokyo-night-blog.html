React HooksでカスタムフックをTypeScriptで実装する

📅 2025年10月4日 👤 yamada_taro ⏱️ 5分で読める

React Hooksを使った開発が主流になってきましたが、カスタムフックを適切にTypeScriptで型付けすることで、 より安全で保守性の高いコードを書くことができます。今回は実践的なカスタムフックの実装例を紹介します。

基本的なカスタムフックの実装

まずは、ローカルストレージと同期するuseLocalStorageフックを実装してみましょう。 このフックは状態管理とストレージへの永続化を同時に行います。

TypeScript 
import { useState, useEffect } from 'react';

function useLocalStorage<T>(
  key: string,
  initialValue: T
): [T, (value: T) => void] {
  // 初期値を取得（ローカルストレージまたはデフォルト値）
  const [storedValue, setStoredValue] = useState<T>(() => {
    try {
      const item = window.localStorage.getItem(key);
      return item ? JSON.parse(item) : initialValue;
    } catch (error) {
      console.error(error);
      return initialValue;
    }
  });

  // 値を設定する関数
  const setValue = (value: T) => {
    try {
      setStoredValue(value);
      window.localStorage.setItem(key, JSON.stringify(value));
    } catch (error) {
      console.error(error);
    }
  };

  return [storedValue, setValue];
}

実際の使用例

上記のカスタムフックは以下のように使用できます。型推論も完璧に機能します。

TypeScript (React) 
interface UserPreferences {
  theme: 'light' | 'dark';
  language: string;
  notifications: boolean;
}

function SettingsComponent() {
  const [preferences, setPreferences] = useLocalStorage<UserPreferences>(
    'userPreferences',
    { 
      theme: 'dark', 
      language: 'ja', 
      notifications: true 
    }
  );

  return (
    <div>
      <button onClick={() => 
        setPreferences({ ...preferences, theme: 'light' })
      }>
        ライトテーマに変更
      </button>
    </div>
  );
}

ポイント

  • ジェネリクスを使用することで、任意の型に対応できる柔軟なフックになります
  • 初期化時にtry-catchでエラーハンドリングを行うことで、堅牢性が向上します
  • TypeScriptの型推論により、使用側で型注釈を省略できる場合もあります
  • 戻り値をタプル型で定義することで、useStateと同じインターフェースを提供できます
💡 カスタムフックを作成する際は、必ずuseプレフィックスをつけることで、 React開発者ツールやlintルールが正しく機能します。
